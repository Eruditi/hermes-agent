# 宝可梦玩家

通过无头模拟器使用`pokemon-agent`包玩宝可梦游戏。

## 何时使用
- 用户说"玩宝可梦"、"启动宝可梦"、"宝可梦游戏"
- 用户询问宝可梦红、蓝、黄、火红等
- 用户想要观看AI玩宝可梦
- 用户引用ROM文件（.gb、.gbc、.gba）

## 启动程序

### 1. 首次设置（克隆、虚拟环境、安装）
仓库是GitHub上的NousResearch/pokemon-agent。克隆它，然后设置Python 3.10+虚拟环境。使用uv（速度优先）创建虚拟环境并以可编辑模式安装包，带有pyboy附加组件。如果uv不可用，回退到python3 -m venv + pip。

在此机器上，它已在/home/teknium/pokemon-agent设置好，虚拟环境已准备就绪 — 只需cd到那里并source .venv/bin/activate。

您还需要一个ROM文件。向用户询问他们的ROM。在此机器上，该目录中的roms/pokemon_red.gb存在一个ROM。
切勿下载或提供ROM文件 — 始终向用户询问。

### 2. 启动游戏服务器
从pokemon-agent目录内部，虚拟环境已激活，运行pokemon-agent serve，使用--rom指向ROM，--port 9876。
在后台运行它，使用&。
要从保存的游戏恢复，添加--load-state与保存名称。
等待4秒启动，然后使用GET /health验证。

### 3. 为用户设置实时仪表板以观看
使用通过localhost.run的SSH反向隧道，以便用户可以在他们的浏览器中查看仪表板。连接ssh，将本地端口9876转发到nokey@localhost.run上的远程端口80。将输出重定向到日志文件，等待10秒，然后grep日志中的.lhr.life URL。给用户附加了/dashboard/的URL。
隧道URL每次更改 — 如果重新启动，给用户新的URL。

## 保存和加载

### 何时保存
- 每15-20回合游戏
- 始终在道馆战斗、劲敌遭遇或危险战斗之前
- 进入新城镇或地牢之前
- 在您不确定的任何行动之前

### 如何保存
使用描述性名称POST /save。好的例子：
before_brock、route1_start、mt_moon_entrance、got_cut

### 如何加载
使用保存名称POST /load。

### 列出可用的保存
GET /saves返回所有保存的状态。

### 服务器启动时加载
启动服务器时使用--load-state标志以自动加载保存。这比启动后通过API加载更快。

## 游戏循环

### 步骤1：观察 — 检查状态并截图
GET /state获取位置、HP、战斗、对话框。
GET /screenshot并保存到/tmp/pokemon.png，然后使用vision_analyze。
始终两者都做 — RAM状态提供数字，视觉提供空间意识。

### 步骤2：定向
- 屏幕上有对话框/文本 → 前进
- 在战斗中 → 战斗或逃跑
- 队伍受伤 → 前往宝可梦中心
- 接近目标 → 小心导航

### 步骤3：决定
优先级：对话框 > 战斗 > 治疗 > 故事目标 > 训练 > 探索

### 步骤4：行动 — 最多移动2-4步，然后重新检查
POST /action使用短行动列表（2-4个行动，不是10-15个）。

### 步骤5：验证 — 每次移动序列后截图
截图并使用vision_analyze确认您移动到了预期的位置。这是最重要的步骤。没有视觉，您会迷路。

## 行动参考
- press_a — 确认、交谈、选择
- press_b — 取消、关闭菜单
- press_start — 打开游戏菜单
- walk_up/down/left/right — 移动一格
- hold_b_N — 按住B N帧（用于快速浏览文本）
- wait_60 — 等待约1秒（60帧）
- a_until_dialog_end — 反复按A直到对话框清除

## 经验中的关键提示

### 持续使用视觉
- 每2-4步移动后截图
- RAM状态告诉您位置和HP，但不告诉您周围有什么
- 窗台、栅栏、标志、建筑门、NPC — 仅通过截图可见
- 向视觉模型询问具体问题："我北边一格是什么？"
- 卡住时，在尝试随机方向之前始终截图

### 传送过渡需要额外等待时间
穿过门或楼梯时，屏幕在地图过渡期间会变黑。您必须等待它完成。在任何门/楼梯传送后添加2-3个wait_60行动。不等待的话，位置读取会过时，您会认为自己仍在旧地图中。

### 建筑退出陷阱
退出建筑时，您会直接出现在门前面。如果向北走，您会直接回到里面。始终先向左右走2格，然后朝您想要的方向前进。

### 对话框处理
第1代文本逐字母缓慢滚动。要快速浏览对话框，按住B 120帧，然后按A。根据需要重复。按住B使文本以最大速度显示。然后按A前进到下一行。
a_until_dialog_end行动检查RAM对话框标志，但此标志不会捕获所有文本状态。如果对话框似乎卡住，请使用手动的hold_b + press_a模式，并通过截图验证。

### 窗台是单向的
窗台（小悬崖边缘）只能向下跳（南），不能向上爬（北）。如果被向北的窗台挡住，您必须向左或向右走以找到绕过它的缺口。使用视觉识别缺口的方向。明确询问视觉模型。

### 导航策略
- 一次移动2-4步，然后截图检查位置
- 进入新区域时，立即截图以定向
- 向视觉模型询问"朝[目的地]哪个方向走？"
- 如果尝试3+次仍卡住，截图并完全重新评估
- 不要发送10-15次移动 — 您会过头或卡住

### 从野生战斗中逃跑
在战斗菜单上，逃跑在右下角。要从默认光标位置（战斗，左上）到达它：按下然后向右移动光标到逃跑，然后按A。用hold_b包裹以快速浏览文本/动画。

### 战斗（战斗）
在战斗菜单上，战斗在左上（默认光标位置）。按A进入招式选择，再次按A使用第一个招式。然后按住B以快速浏览攻击动画和文本。

## 战斗策略

### 决策树
1. 想要捕捉？ → 削弱然后扔精灵球
2. 您不需要的野生？ → 逃跑
3. 属性优势？ → 使用效果拔群的招式
4. 无优势？ → 使用最强的本系招式
5. 低HP？ → 切换或使用药水

### 第1代属性图表（关键配对）
- 水克火、地面、岩石
- 火克草、虫、冰
- 草克水、地面、岩石
- 电克水、飞行
- 地面克火、电、岩石、毒
- 超能力克格斗、毒（第1代占主导！）

### 第1代怪癖
- 特殊属性 = 特殊招式的攻击和防御
- 超能力属性过强（幽灵招式有bug）
- 暴击基于速度属性
- 缠绕/绑定防止对手行动
- 聚气bug：降低暴击率而不是提高它

## 停止游戏
1. 通过POST /save使用描述性名称保存游戏
2. 使用PKM:PROGRESS更新内存
3. 告诉用户："游戏保存为[名称]！说'玩宝可梦'以恢复。"
4. 终止服务器和隧道后台进程

## 陷阱
- 切勿下载或提供ROM文件
- 不要发送超过4-5个行动而不检查视觉
- 退出建筑后向北走之前始终向旁边走
- 始终在门/楼梯传送后添加wait_60 x2-3
- 通过RAM的对话框检测不可靠 — 使用截图验证
- 在危险遭遇之前保存
- 每次重新启动时隧道URL都会更改
