# 歌曲创作与AI音乐生成

这里的一切都是指导方针，不是规则。艺术故意打破规则。使用对歌曲有用的东西。忽略没用的。

---

## 1. 歌曲结构（选择一个或自己发明）

常见骨架 — 根据需要混合、修改或丢弃：

```
ABABCB  主歌/副歌/主歌/副歌/桥接/副歌    (大多数流行/摇滚)
AABA    主歌/主歌/桥接/主歌（基于副歌）    (爵士标准，民谣)
ABAB    主歌/副歌交替                    (简单，直接)
AAA     主歌/主歌/主歌（分节，无副歌）     (民谣，讲故事)
```

六个构建块：
-  intro      — 设定情绪，吸引听众
-  verse      — 故事，细节，世界构建
-  pre-chorus — 可选的紧张感上升，为高潮做准备
-  chorus     — 情感核心，人们记住的部分
-  bridge     — 绕道，视角或调性的转变
-  outro      — 告别，可以呼应或颠覆其余部分

你不需要所有这些。一些伟大的歌曲只是一个不断演变的部分。结构服务于情感，而不是相反。

## 2. 押韵、节奏和声音

押韵类型（从紧密到松散）：
- 完美：lean/mean
- 家族：crate/braid
- 元音：had/glass（相同元音，不同结尾）
- 辅音：scene/when（不同元音，相似结尾）
- 近/斜：足以暗示联系而不锁定

混合使用它们。全是完美押韵听起来像儿歌。全是斜押韵听起来懒惰。混合才是关键。

内部押韵：在行内押韵，不仅仅在结尾。
  "We pruned the lies from bleeding trees / Distilled the storm
   from entropy" — "lies/flies," "trees/entropy" 创建内部回声。

节奏：重音与非重音音节的节奏。
- 平行行之间匹配音节数有助于可唱性
- 重音音节比总计数更重要
- 大声说出来。如果你结巴，节奏需要工作。
- 故意打破节奏可以创造强调或惊喜

---

## 3. 情感弧线和动态

把歌曲看作一段旅程，不是平坦的道路。

能量映射（大致想法，不是处方）：
  引子：2-3  |  主歌：5-6  |  预副歌：7
  副歌：8-9  |  桥接：变化  |  最终副歌：9-10

最强大的动态技巧：对比。
- 尖叫前的耳语比只是尖叫更有力
- 稀疏前的密集。缓慢前的快速。低前的高。
- 下降只有因为有上升才有效
- 沉默是一种乐器

"从耳语到咆哮到耳语" — 从亲密开始，构建到全力，
回到脆弱。适用于民谣，史诗， anthem。

## 4. 创作有效的歌词

展示，不要告诉（通常）：
- "我很伤心" = 平淡
- "你的连帽衫还挂在门旁的钩子上" = 生动
- 但有时"我付出我的生命"直白地说就是力量

钩子：
- 人们记住、哼唱、重复的线
- 通常是标题或核心短语
- 当旋律+歌词+情感都对齐时效果最好
- 放在最有力的地方（通常是副歌的第一/最后一行）

韵律 — 歌词和音乐相互支持：
- 稳定的感觉（解决，和平）与稳定的旋律、
  完美的押韵、解决的和弦配对
- 不稳定的感觉（渴望，怀疑）与徘徊的旋律、
  近押韵、未解决的和弦配对
- 主歌旋律通常较低，副歌较高
- 但如果对歌曲有帮助，就翻转它

避免（除非你是故意的）：
- 自动模式的陈词滥调（"金子般的心"没有赢得它）
- 为了押韵而强迫词序（"尤达语"）
- 每个部分都有相同的能量（平淡的动态）
- 把你的初稿视为神圣 — 修改是创造

## 5. 恶搞和改编

当用新歌词重写现有歌曲时：

骨架：首先映射原作的结构。
- 计算每行的音节数
- 标记押韵方案（ABAB，AABB等）
- 识别哪些音节是重音
- 注意长音/持续音落在哪里

适应新单词：
- 将重音音节与原作的相同节拍匹配
- 总音节数可以灵活1-2个非重音音节
- 在长音上，尝试匹配原作的元音声音
  （如果原作以"LOOOVE"的"oo"元音保持，"FOOOD"比"LIFE"更合适）
- 关键位置的单音节交换保持节奏完整
  （Crime -> Code, Snake -> Noose）
- 用原曲唱你的新单词 — 如果你结巴，就修改

概念：
- 选择足够强的概念来支撑整首歌
- 从标题/钩子开始，向外构建
- 首先生成大量原始材料（双关语，短语，图像），
  然后将最好的放入结构
- 如果你在某处需要特定的行，反向工程
  押韵方案来设置它

保留一些原作：保留一些原始行或结构
完整增加可识别性，让观众感受到联系。

## 6. Suno AI提示工程

### 风格/流派描述字段

公式（根据需要调整）：
  流派 + 情绪 + 时代 + 乐器 +  vocal style + 制作 + 动态

```
差：  "悲伤的摇滚歌曲"
好： "电影管弦乐间谍惊悚片，1960年代冷战时代，烟雾缭绕的
       性感女性 vocalist，大乐队爵士乐，带有
       小号和法国号的铜管部分，横扫弦乐，小调，
       复古模拟温暖"
```

描述旅程，而不仅仅是流派：
```
"以稀疏钢琴上的 haunting 耳语开始。逐渐分层
  静音铜管。通过副歌与完整管弦乐构建。
  第二节爆发原始的belting强度。结尾剥离
  回到孤独的钢琴和脆弱的耳语，渐弱至沉默。"
```

提示：
- V4.5+支持风格字段中最多1,000个字符 — 使用它们
- 不要使用艺术家名称或商标。而是描述声音。
  "1960年代冷战间谍惊悚片铜管" 而不是 "James Bond风格"
  "90年代垃圾摇滚" 而不是 "Nirvana风格"
- 当你有偏好时指定BPM和调
- 使用排除风格字段排除你不想要的
- 意外的流派组合可能是黄金："巴萨诺瓦陷阱"，
  "阿巴拉契亚哥特式"，"芯片音乐爵士"
- 构建 vocal 角色，而不仅仅是性别：
  "一个饱经风霜的 torch singer，烟熏女低音，轻微沙哑，
   从脆弱开始，构建到毁灭性的力量"

### 元标签（放在歌词字段内的[括号]中）

结构：
  [Intro] [Verse] [Verse 1] [Pre-Chorus] [Chorus]
  [Post-Chorus] [Hook] [Bridge] [Interlude]
  [Instrumental] [Instrumental Break] [Guitar Solo]
  [Breakdown] [Build-up] [Outro] [Silence] [End]

vocal 表演：
  [Whispered] [Spoken Word] [Belted] [Falsetto] [Powerful]
  [Soulful] [Raspy] [Breathy] [Smooth] [Gritty]
  [Staccato] [Legato] [Vibrato] [Melismatic]
  [Harmonies] [Choir] [Harmonized Chorus]

动态：
  [High Energy] [Low Energy] [Building Energy] [Explosive]
  [Emotional Climax] [Gradual swell] [Orchestral swell]
  [Quiet arrangement] [Falling tension] [Slow Down]

性别：
  [Female Vocals] [Male Vocals]

氛围：
  [Melancholic] [Euphoric] [Nostalgic] [Aggressive]
  [Dreamy] [Intimate] [Dark Atmosphere]

音效：
  [Vinyl Crackle] [Rain] [Applause] [Static] [Thunder]

在风格字段和歌词中都放置标签以加强。
每个部分最多保持5-8个标签 — 太多会混淆AI。
不要自相矛盾（同一部分中的[Calm] + [Aggressive]）。

### 自定义模式
- 认真工作时始终使用自定义模式（分开的风格+歌词）
- 歌词字段限制：~3,000字符（~40-60行）
- 始终添加结构标签 — 没有它们，Suno默认
  扁平的主歌/副歌/主歌，没有情感弧线

## 7. AI歌手的语音技巧

AI vocalists 不阅读 — 他们发音。帮助他们：

语音重拼：
- 按发音拼写单词："through" -> "thru"
- 专有名词失败率最高 — 早期测试
- "Nous" -> "Noose"（强制正确发音）
- 用连字符引导音节："Re-search", "bio-engineering"

交付控制：
- 全大写 = 更响亮，更强烈
- 元音延长："lo-o-o-ove" = 持续/ melisma
- 省略号："I... need... you" = 戏剧性停顿
- 连字符拉伸："ne-e-ed" = 情感拉伸

始终：
- 拼出数字："24/7" -> "twenty four seven"
- 空格缩写："AI" -> "A I" 或 "A-I"
- 首先在30秒短片段中测试专有名词/不寻常单词
- 生成后，发音被烘焙 — 在歌词中修复之前

## 8. 工作流程

1. 首先写概念/钩子 — 情感核心是什么？
2. 如果改编，映射原始结构（音节，押韵，重音）
3. 生成原始材料 — 在结构化之前自由头脑风暴
4. 将歌词草稿放入结构
5. 大声朗读/唱歌 — 捕捉结巴，修复节奏
6. 构建Suno风格描述 — 描绘动态旅程
7. 为表演方向添加元标签到歌词
8. 生成至少3-5个变体 — 将它们视为录音
9. 选择最好的，使用Extend/Continue构建有前途的部分
10. 如果意外发生了很棒的事情，保留它

预期：每1个好结果约3-5次生成。修改是正常的。
扩展时风格可能漂移 — 扩展时重述流派/情绪。

---

## 9. 经验教训

- 在风格字段中描述动态弧线比仅仅列出流派重要得多。"从耳语到咆哮到耳语"给
  Suno一个表演地图。
- 在恶搞中保留一些原始行增加可识别性
  和情感重量 — 观众感受到原作的幽灵。
- 歌曲中的桥接位置是你可以转换意象的地方。
  交换原作的具体参考为你的主题的隐喻，
  同时保持情感功能（反思，转变，启示）。
- 钩子/标签中的单音节单词交换是最干净的方式
  在改变意义的同时保持节奏。
- 风格字段中强大的vocal角色描述比任何单个元标签都更重要。
- 不要对规则太在意。如果一行打破节奏但更有力，保留它。感觉才是重要的。技巧服务于艺术，
  而不是相反。
