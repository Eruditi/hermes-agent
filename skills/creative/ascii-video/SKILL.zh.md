# ASCII视频生产管道

## 创意标准

这是视觉艺术。ASCII字符是媒介；电影是标准。

**在编写第一行代码之前**，阐明创意概念。情绪是什么？这讲述了什么视觉故事？是什么让这个项目与其他每个ASCII视频不同？用户的提示是一个起点 — 用创意雄心来解释它，而不是字面转录。

**第一次渲染卓越是不可谈判的。** 输出必须在视觉上引人注目，无需修订轮次。如果某些东西看起来通用、平淡或像"AI生成的ASCII艺术"，那就是错误的 — 在交付之前重新思考创意概念。

**超越参考词汇。** 参考中的效果目录、着色器预设和调色板库是一个起始词汇。对于每个项目，组合、修改和发明新模式。目录是颜料调色板 — 您来写画。

**积极主动地创意。** 当项目需要时，扩展技能的词汇。如果参考资料没有视觉需求的内容，就构建它。包括至少一个用户没有要求但会欣赏的视觉时刻 — 过渡、效果、提升整个作品的颜色选择。

**凝聚力美学胜于技术正确性。** 视频中的所有场景必须通过统一的视觉语言联系起来 — 共享色温、相关字符调色板、一致的运动词汇。一个技术上正确的视频，每个场景使用随机不同的效果，是一个美学失败。

**密集、分层、深思熟虑。** 每一帧都应该奖励观看。永远不要平坦的黑色背景。始终多网格构图。始终每个场景变化。始终有意的颜色。

## 堆栈

每个项目单个自包含的Python脚本。无需GPU。

| 层 | 工具 | 目的 |
|-------|------|---------|
| 核心 | Python 3.10+, NumPy | 数学、数组操作、向量化效果 |
| 信号 | SciPy | FFT、峰值检测（音频模式） |
| 成像 | Pillow (PIL) | 字体光栅化、帧解码、图像I/O |
| 视频I/O | ffmpeg (CLI) | 解码输入、编码输出、复用音频 |
| 并行 | concurrent.futures | N个工作器用于批量/剪辑渲染 |
| TTS | ElevenLabs API（可选） | 生成叙述剪辑 |
| 可选 | OpenCV | 视频帧采样、边缘检测 |

## 管道架构

每个模式都遵循相同的6阶段管道：

```
输入 → 分析 → 场景函数 → 色调映射 → 着色 → 编码
```

1. **输入** — 加载/解码源素材（视频帧、音频样本、图像或无）
2. **分析** — 提取每帧特征（音频带、视频亮度/边缘、运动向量）
3. **场景函数** — 场景函数渲染到像素画布（`uint8 H,W,3`）。通过`_render_vf()` + 像素混合模式组合多个字符网格。参见`references/composition.md`
4. **色调映射** — 基于百分位数的自适应亮度归一化。参见`references/composition.md` § 自适应色调映射
5. **着色** — 通过`ShaderChain` + `FeedbackBuffer`进行后处理。参见`references/shaders.md`
6. **编码** — 将原始RGB帧管道到ffmpeg进行H.264/GIF编码

### 每节变化

永远不要对整个视频使用相同的配置。对于每个节/场景：
- **不同的背景效果**（或组合2-3个）
- **不同的字符调色板**（匹配情绪）
- **不同的颜色策略**（或者至少不同的色调）
- **变化着色器强度**（峰值期间更多绽放，安静期间更多颗粒）
- **不同的粒子类型**（如果粒子处于活动状态）

## 工作流程

### 步骤1：创意愿景

在任何代码之前，阐明创意概念：

- **情绪/氛围**：观众应该感觉到什么？充满活力、冥想、混乱、优雅、不祥？
- **视觉故事**：在持续时间内发生了什么？建立紧张感？转变？溶解？
- **颜色世界**：温暖/凉爽？单色？霓虹？土色调？主色调是什么？
- **字符纹理**：密集数据？稀疏星星？有机点？几何块？
- **是什么让这个不同**：是什么让这个项目独一无二的一件事？
- **情感弧线**：场景如何进展？充满活力地开放，建立到高潮，解决？

将用户的提示映射到美学选择。一个"chill lo-fi可视化器"需要与"故障赛博朋克数据流"完全不同的一切。

### 步骤2：技术设计

- **模式** — 上述6种模式中的哪一种
- **分辨率** — 横向1920x1080（默认）、纵向1080x1920、方形1080x1080 @ 24fps
- **硬件检测** — 自动检测核心/RAM，设置质量配置文件。参见`references/optimization.md`
- **节** — 将时间戳映射到场景函数，每个都有自己的效果/调色板/颜色/着色器配置
- **输出格式** — MP4（默认）、GIF（640x360 @ 15fps）、PNG序列

### 步骤3：构建脚本

单个Python文件。组件（带参考）：

1. **硬件检测 + 质量配置文件** — `references/optimization.md`
2. **输入加载器** — 模式相关；`references/inputs.md`
3. **特征分析器** — 音频FFT、视频亮度或合成
4. **网格 + 渲染器** — 带有位图缓存的多密度网格；`references/architecture.md`
5. **字符调色板** — 每个项目多个；`references/architecture.md` § 调色板
6. **颜色系统** — HSV + 离散RGB + 和声生成；`references/architecture.md` § 颜色
7. **场景函数** — 每个返回`canvas (uint8 H,W,3)`；`references/scenes.md`
8. **色调映射** — 自适应亮度归一化；`references/composition.md`
9. **着色器管道** — `ShaderChain` + `FeedbackBuffer`；`references/shaders.md`
10. **场景表 + 调度器** — 时间 → 场景函数 + 配置；`references/scenes.md`
11. **并行编码器** — 带ffmpeg管道的N工作器剪辑渲染
12. **主函数** — 协调完整管道

### 步骤4：质量验证

- **首先测试帧**：在完整渲染之前在关键时间戳渲染单个帧
- **亮度检查**：所有ASCII内容的`canvas.mean() > 8`。如果暗，降低伽马
- **视觉连贯性**：所有场景都感觉属于同一个视频吗？
- **创意愿景检查**：输出与步骤1的概念匹配吗？如果看起来通用，请返回

## 关键实现说明

### 亮度 — 使用`tonemap()`，而不是线性乘法器

这是#1视觉问题。黑色上的ASCII本质上是暗的。**永远不要使用`canvas * N`乘法器** — 它们裁剪高光。使用自适应色调映射：

```python
def tonemap(canvas, gamma=0.75):
    f = canvas.astype(np.float32)
    lo, hi = np.percentile(f[::4, ::4], [1, 99.5])
    if hi - lo < 10: hi = lo + 10
    f = np.clip((f - lo) / (hi - lo), 0, 1) ** gamma
    return (f * 255).astype(np.uint8)
```

管道：`scene_fn() → tonemap() → FeedbackBuffer → ShaderChain → ffmpeg`

每场景伽马：默认0.75，日晒0.55，色调分离0.50，明亮场景0.85。对暗层使用`screen`混合（不是`overlay`）。

### 字体单元格高度

macOS Pillow：`textbbox()`返回错误的高度。使用`font.getmetrics()`：`cell_height = ascent + descent`。参见`references/troubleshooting.md`。

### ffmpeg管道死锁

永远不要将`stderr=subprocess.PIPE`与长时间运行的ffmpeg一起使用 — 缓冲区在64KB填充并死锁。重定向到文件。参见`references/troubleshooting.md`。

### 字体兼容性

并非所有Unicode字符都在所有字体中渲染。在初始化时验证调色板 — 渲染每个字符，检查空白输出。参见`references/troubleshooting.md`。

### 每剪辑架构

对于分段视频（引用、场景、章节），将每个渲染为单独的剪辑文件，以进行并行渲染和选择性重新渲染。参见`references/scenes.md`。

## 创意分歧（仅当用户请求实验性/创意/独特输出时使用）

如果用户要求创意、实验性、令人惊讶或非传统的输出，请选择最适合的策略，并在生成代码之前推理其步骤。

- **强制连接** — 当用户想要跨领域灵感时（"让它看起来有机"，"工业美学"）
- **概念混合** — 当用户命名两个要组合的东西时（"海洋遇见音乐"，"太空+书法"）
- **倾斜策略** — 当用户最大程度开放时（"给我惊喜"，"我从未见过的东西"）

### 强制连接
1. 选择一个与视觉目标无关的领域（天气系统、微生物学、建筑、流体动力学、纺织编织）
2. 列出其核心视觉/结构元素（侵蚀 → 逐渐揭示；有丝分裂 → 分裂复制；编织 → 互锁模式）
3. 将这些元素映射到ASCII字符和动画模式
4. 合成 — "侵蚀"或"结晶"在字符网格中看起来像什么？

### 概念混合
1. 命名两个不同的视觉/概念空间（例如，海浪 + 乐谱）
2. 映射对应关系（波峰 = 高音，波谷 = 休止，泡沫 = 断奏）
3. 有选择地混合 — 保持最有趣的映射，丢弃强制的映射
4. 发展仅存在于混合中的新兴属性

### 倾斜策略
1. 画一个："尊重你的错误作为隐藏的意图" / "使用一个旧想法" / "你最亲密的朋友会做什么？" / "强调缺陷" / "把它颠倒过来" / "只有一部分，不是全部" / "反转"
2. 根据当前ASCII动画挑战解释指令
3. 在编写代码之前将横向洞察力应用于视觉设计
