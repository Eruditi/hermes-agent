# 创意构思

通过创意约束生成项目想法。约束 + 方向 = 创造力。

## 工作原理

1. **选择一个约束** 从下面的库中选择 — 随机选择，或根据用户的领域/心情匹配
2. **广泛解释它** — 一个编程提示可以变成一个硬件项目，一个艺术提示可以变成一个CLI工具
3. **生成3个具体的项目想法** 满足约束
4. **如果他们选择一个，就构建它** — 创建项目，编写代码，发布它

## 规则

每个提示都被尽可能广泛地解释。"这包括X吗？" → 是的。提示提供方向和轻微的约束。没有这些，就没有创造力。

## 约束库

### 对于开发者

**解决你自己的痒处：**
构建你本周希望存在的工具。50行以内。今天就发布它。

**自动化烦人的事情：**
你工作流程中最乏味的部分是什么？用脚本把它去掉。花两个小时来修复一个每天花费你五分钟的问题。

**应该存在的CLI工具：**
想想一个你希望可以输入的命令。`git undo-that-thing-i-just-did`。`docker why-is-this-broken`。`npm explain-yourself`。现在构建它。

**除了胶水什么都没有新的：**
完全用现有的API、库和数据集制作东西。唯一的原创贡献是你如何连接它们。

**科学怪人周：**
拿一个做X的东西，让它做Y。一个播放音乐的git仓库。一个生成诗歌的Dockerfile。一个发送赞美的cron作业。

**减法：**
在代码库崩溃之前，你能删除多少？把一个工具剥离到它的最小可行功能。删除直到只剩下本质。

**高概念，低努力：**
一个深刻的想法，懒惰地执行。概念应该是精彩的。实现应该花一个下午。如果花更长时间，你想太多了。

### 对于制作者和艺术家

**公然复制一些东西：**
选择你钦佩的东西 — 一个工具，一件艺术品，一个界面。从头开始重新创建它。学习在你的版本和他们的版本之间的差距中。

**一百万个什么：**
一百万既是很多，也不是那么多。一百万像素是一张1MB的照片。一百万次API调用是一个星期二。任何东西的一百万在规模上变得有趣。

**制作一些会消亡的东西：**
一个每天失去一个功能的网站。一个会忘记的聊天机器人。一个倒数到虚无的倒计时。一个关于腐烂、杀戮或放手的练习。

**做很多数学：**
生成几何，着色器高尔夫，数学艺术，计算折纸。是时候重新学习什么是arcsin了。

### 对于任何人

**文本是通用界面：**
构建一个文本是唯一界面的东西。没有按钮，没有图形，只有文字进和文字出。文本几乎可以进出任何东西。

**从笑点开始：**
想想一些会是有趣句子的东西。倒推使其成为现实。"我教我的恒温器对我进行煤气灯操纵" → 现在构建它。

**敌对的UI：**
制作一些故意让人痛苦使用的东西。一个需要47个条件的密码字段。一个每个标签都撒谎的表单。一个评判你命令的CLI。

**再来一次：**
记住一个旧项目。从头再做一次。不要看原版。看看你的思维方式有什么变化。

查看 `references/full-prompt-library.md` 以获取30多个额外的约束，涵盖沟通、规模、哲学、转换等。

## 将约束与用户匹配

| 用户说 | 从哪里选择 |
|--------|-----------|
| "我想构建一些东西"（没有方向） | 随机 — 任何约束 |
| "我正在学习[语言]" | 公然复制一些东西，自动化烦人的事情 |
| "我想要一些奇怪的东西" | 敌对的UI，科学怪人周，从笑点开始 |
| "我想要一些有用的东西" | 解决你自己的痒处，应该存在的CLI工具，自动化烦人的事情 |
| "我想要一些美丽的东西" | 做很多数学，一百万个什么 |
| "我累坏了" | 高概念低努力，制作一些会消亡的东西 |
| "周末项目" | 除了胶水什么都没有新的，从笑点开始 |
| "我想要一个挑战" | 一百万个什么，减法，再来一次 |

## 输出格式

```
## 约束：[名称]
> [约束，一句话]

### 想法

1. **[一句话宣传语]**
   [2-3句话：你会构建什么以及为什么它有趣]
   ⏱ [周末 / 周 / 月] • 🔧 [技术栈]

2. **[一句话宣传语]**
   [2-3句话]
   ⏱ ... • 🔧 ...

3. **[一句话宣传语]**
   [2-3句话]
   ⏱ ... • 🔧 ...
```

## 示例

```
## 约束：应该存在的CLI工具
> 想想一个你希望可以输入的命令。现在构建它。

### 想法

1. **`git whatsup` — 显示你不在时发生了什么**
   将你最后一次活跃的提交与HEAD进行比较，总结发生了什么变化，
   谁提交了，以及合并了哪些PR。就像你的仓库的早晨站立会议。
   ⏱ 周末 • 🔧 Python, GitPython, click

2. **`explain 503` — 人类可读的HTTP状态码**
   管道输入任何状态码或错误消息，获取带有常见原因和修复的纯英文解释。
   从策划的数据库中提取，而不是LLM。
   ⏱ 周末 • 🔧 Rust或Go，静态数据集

3. **`deps why <package>` — 为什么这个在我的依赖树中**
   追溯一个传递依赖回拉入它的直接依赖。
   用一个命令回答"为什么我有47份lodash"。
   ⏱ 周末 • 🔧 Node.js, npm/yarn锁文件解析
```

用户选择一个后，开始构建 — 创建项目，编写代码，迭代。
